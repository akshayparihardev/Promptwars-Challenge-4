# 08 — Database Schema

## Executive Summary

**WHY:** The Live Stadium State is persisted here; schema quality determines reasoning quality and auditability.

**Engine:** SQLite (file-based, zero-ops, < 10MB friendly). **ORM:** Prisma. Migrations checked in.

---

## Entity-Relationship Diagram

```
OperationalEvent ──< references >── (evidence) ──> Recommendation
Recommendation ──1:1──> Decision ──1:1──> Action ──> OutcomeEvent(OperationalEvent)
Recommendation ──1:many──> RecommendationLocalization
Recommendation ──1:0..1──> OperationalMemory (post-execution accuracy)
Situation ──1:many──> Recommendation
HealthSnapshot (periodic, per reasoning cycle)
All state changes ──> AuditLog
```

## Prisma Schema

```prisma
// prisma/schema.prisma
generator client { provider = "prisma-client-js" }
datasource db { provider = "sqlite"; url = env("DATABASE_URL") }

enum Domain {
  navigation crowd transport accessibility
  sustainability multilingual operations medical security
}
enum Severity { low medium high critical }
enum RecStatus { proposed approved rejected executed expired }
enum Role {
  fan volunteer security medical organizer
  venue_operations accessibility_coordinator transportation_coordinator
}

model OperationalEvent {
  id          String   @id @default(cuid())
  domain      Domain
  zone        String                    // e.g., "gate-7", "concourse-north"
  type        String                    // e.g., "density_reading","incident"
  severity    Severity @default(low)
  payload     String                    // JSON string (validated on write)
  isOutcome   Boolean  @default(false)  // true if generated by an Action
  createdAt   DateTime @default(now())

  @@index([domain, createdAt])
  @@index([zone, createdAt])
}

model Situation {
  id              String   @id @default(cuid())
  title           String
  domains         String                 // JSON array
  severity        Severity
  rationale       String
  signature       String   @unique       // dedup key
  createdAt       DateTime @default(now())
  recommendations Recommendation[]
}

model Recommendation {
  id               String   @id @default(cuid())
  situationId      String
  situation        Situation @relation(fields: [situationId], references: [id])
  domain           Domain
  targetRoles      String                // JSON array of Role
  title            String
  recommendedAction String
  reason           String
  evidence         String                // JSON array of EvidenceRef
  confidence       Float
  expectedImpact   String                // JSON object
  alternatives     String                // JSON array of Alternative[]
  prediction       String                // JSON object: { noActionOutcome, predictedMetrics, healthScoreImpact }
  priority         Float
  status           RecStatus @default(proposed)
  source           String                // "genai" | "deterministic"
  createdAt        DateTime @default(now())
  expiresAt        DateTime
  decision         Decision?
  localizations    RecommendationLocalization[]
  memory           OperationalMemory?

  @@index([status, priority])
  @@index([domain, status])
}

model RecommendationLocalization {
  id               String @id @default(cuid())
  recommendationId String
  recommendation   Recommendation @relation(fields: [recommendationId], references: [id])
  lang             String
  title            String
  reason           String
  expectedImpact   String
  @@unique([recommendationId, lang])
}

model Decision {
  id               String   @id @default(cuid())
  recommendationId String   @unique
  recommendation   Recommendation @relation(fields: [recommendationId], references: [id])
  actorRole        Role
  outcome          String                // "approved" | "rejected"
  note             String?
  createdAt        DateTime @default(now())
  action           Action?
}

model Action {
  id          String   @id @default(cuid())
  decisionId  String   @unique
  decision    Decision @relation(fields: [decisionId], references: [id])
  effect      String                     // JSON: simulated effect
  outcomeEventId String?                 // link to generated feedback event
  createdAt   DateTime @default(now())
}

model AuditLog {
  id         String   @id @default(cuid())
  entityType String                      // "recommendation"|"decision"|"action"|"event"
  entityId   String
  action     String                      // "created"|"approved"|"rejected"|"executed"|"expired"
  actorRole  Role?
  metadata   String                      // JSON
  createdAt  DateTime @default(now())
  @@index([entityType, entityId])
  @@index([createdAt])
}

model OperationalMemory {
  id                String   @id @default(cuid())
  recommendationId  String   @unique
  recommendation    Recommendation @relation(fields: [recommendationId], references: [id])
  situationSignature String                 // same dedup key as Situation
  domain            Domain
  predictedMetrics  String                  // JSON: Record<string, number>
  actualMetrics     String                  // JSON: Record<string, number>
  predictionAccuracy Float                  // 0..1
  createdAt         DateTime @default(now())
  @@index([domain, createdAt])
  @@index([situationSignature])
}

model HealthSnapshot {
  id         String   @id @default(cuid())
  domains    String                         // JSON: Record<Domain, number> (0-100)
  overall    Float                          // 0-100
  trend      String                         // "improving" | "stable" | "declining"
  createdAt  DateTime @default(now())
  @@index([createdAt])
}
```

## Design Notes (WHY)

- **JSON-as-string columns:** SQLite lacks native JSON typing; we validate with Zod at the application boundary. Keeps schema simple, avoids extra tables for flexible payloads.
- **`alternatives` and `prediction` on Recommendation:** stored as JSON strings; validated by Zod schema at write time. Enables the extended Decision Intelligence fields without schema bloat.
- **`signature` on Situation:** deterministic dedup (hash of domain+zone+type bucket) prevents recommendation spam.
- **`OperationalMemory` linked to Recommendation:** one-to-one after execution; stores prediction vs. actual metrics for the feedback loop.
- **`HealthSnapshot` append-only:** one row per reasoning cycle; enables trend detection over time.
- **AuditLog append-only:** never updated/deleted → immutability by convention + no update use case exposed.
- **Indexes:** tuned for the hot queries — feed (`status`, `priority`), context window (`domain`/`zone`, `createdAt`), memory lookup (`situationSignature`), health trend (`createdAt`).

---

## Data Integrity Rules

- Evidence event IDs **must exist** (checked in application layer before persist).
- Recommendation `expiresAt` = `createdAt` + `REC_TTL_MIN`.
- Decision → Action only when `outcome === 'approved'`.
